Evals run end-to-end: an agent receives a task, source data, and a wiki instance, then produces pages. Output is graded automatically against a rubric.

## Protocol

1. A fresh MediaWiki instance is provisioned with templates and no content pages.
2. **Phase 1 (source)** — the agent snapshots source directories and documents them as `Source:` pages. Graded at checkpoint 1.
3. **Checkpoint gate** — if checkpoint 1 composite < 0.7, Phase 2 is skipped.
4. **Phase 2 (content)** — the agent reads source pages, queries the vault, and writes content pages. Graded at checkpoint 2.
5. **Composite** = 20% checkpoint 1 + 80% checkpoint 2.

The `--from-result` flag reuses checkpoint 1 source pages from a previous run, skipping Phase 1.

## Graders

**Completeness** — checks structural elements: lead paragraph, infobox, body sections, References/Bibliography sections, categories, word count (≥ 800), subsections (≥ 3), inline citations (≥ 10), blockquotes, media embeds. Binary per element with weighted penalties.

**Citations** — validates citation templates (`{{Cite message}}`, `{{Cite vault}}`, etc.) have required fields: `snapshot`, `date`, and type-specific fields. Score = valid / total.

**Accuracy** — resolves citations against vault manifests and checks whether claimed facts are supported by source data. Score = supported / total claims.

**Structure** — evaluates Wikipedia conventions: third-person neutral tone, past tense, section organization, wikitext syntax, wikilinks. Deductions for tone violations (−0.1 each, capped −0.4), structural issues (−0.15), syntax errors (−0.1).

**Cross-referencing** — measures whether the page combines information from multiple source types. Count of cross-referenced facts / possible cross-references. Only scored on multi-source tasks.

**Reference coverage** — compares against a human-written reference page: section heading overlap, infobox field coverage, citation hash coverage, category coverage. Weighted average.

**Tool usage** — checks harness logs for expected CLI usage: `wai snapshot`, `wai read`, `wai write`/`create`/`edit`. Binary per tool.

### Observations

- **Accuracy is the weakest grader across all harnesses.** Claude Code leads at 0.245; Kimi K2.5 is close at 0.312. The eval wiki's vault is populated but citation resolution is still lossy.
- **OpenCode achieves perfect source scores.** Both Kimi K2.5 and MiniMax M2.5 score 1.000 on checkpoint 1, exceeding Claude Code (0.829) and Codex (0.818) on source documentation.
- **Structure separates Claude Code from the rest.** Claude Code scores 0.900 on structure vs 0.450 for Kimi K2.5 and 0.100 for MiniMax M2.5. This reflects prose quality, section organization, and editorial tone.
- **Codex has the strongest citation formatting.** Codex scores 1.000 on citations across its best runs. It produces more citations and uses templates more consistently.
- **Research depth matters more than extraction.** Adding research guidance to the content prompt (search for biographical keywords, sample across timelines, extract quotes) improved Codex's content score from 0.417 to 0.620.

## Known issues

- **Accuracy grader underreports**: citation resolution against vault manifests loses some valid claims as "unsupported."
- **Single-source and interactive benchmarks not yet run**: test cases are defined but only multi-source has results.
