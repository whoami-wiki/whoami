Common issues and how to fix them.

## Wiki not starting

**Symptom**: `wai auth status` shows the wiki as offline, or `http://localhost:8080` is unreachable.

**Fix**: Make sure Docker is running, then restart the wiki from the desktop app or by restarting the Docker container directly:

```bash
docker restart projectwiki
```

If the port is already in use, you can change it in `~/.projectwiki/config.json`:

```json
{ "wiki": { "port": 8081 } }
```

Then restart the wiki for the change to take effect.

## CLI not found in PATH

**Symptom**: Running `wai` returns "command not found."

**Fix**: The desktop app installer should add `wai` to your PATH automatically. If it didn't, add it manually:

```bash
# macOS / Linux
export PATH="$HOME/.projectwiki/bin:$PATH"
```

Add this line to your shell profile (`~/.zshrc`, `~/.bashrc`) to make it permanent.

## Agent harness failing to connect

**Symptom**: The agent can't find ProjectWiki tools or returns connection errors when trying to use them.

**Fix**: Verify the CLI can reach the wiki:

```bash
wai auth status
```

If the plugin is installed but not detected, try removing and re-adding it:

```bash
claude plugin remove projectwiki
claude plugin add projectwiki
```

Make sure you restart your agent session after reinstalling the plugin.

## Permission errors on ingest

**Symptom**: `wai ingest volume` fails with permission denied errors.

**Fix**: Ensure the document directory is readable:

```bash
ls -la ~/project/drawings/
```

On macOS, you may need to grant terminal access to the directory in System Settings → Privacy & Security → Files and Folders.

## Ingest is slow or uses too much memory

**Symptom**: `wai ingest volume` takes a very long time or your machine runs out of memory with large document sets.

**Fix**: Process smaller batches at a time instead of one large volume:

```bash
wai ingest volume ~/project/drawings/civil/ --type drawings --name "Civil Drawings"
wai ingest volume ~/project/drawings/structural/ --type drawings --name "Structural Drawings"
```

## Still stuck?

Check your connection status and include the output when reporting an issue:

```bash
wai auth status --json
```

File issues at [github.com/arborist-ai/projectwiki/issues](https://github.com/arborist-ai/projectwiki/issues).
