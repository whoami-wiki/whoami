title: Evals Suite
description: "Protocol, graders, and scoring methodology for agent evals"
---

Each eval is fully automated: the agent receives a task, contract documents, and a fresh MediaWiki instance, then builds pages across six checkpoints. Output is graded against rubrics and a human-written reference. Accuracy scores are **calibrated**: false penalties are removed where the grader penalized correct facts due to misnumbered citation targets, Talk page editorial notes misclassified as unsupported claims, or partial evidence matches where the core fact is confirmed. Composites are recomputed from calibrated accuracy.

For results across harness and model combinations, see [Choosing a Harness and Model](/docs/choosing-harness-and-model).

## Protocol

Each eval uses a six-checkpoint workflow that mirrors how a project team would build a wiki from contract documents.

1. **Ingest**: Ingest a document volume and create index entries. Catalog drawings by discipline, identify specification sections, assess document quality and gaps. Plan the analysis approach.
2. **Analyze**: Analyze drawings and specifications to build area and equipment pages. Include infoboxes, lead paragraphs, discipline sections with prose and citations. Note gaps and coordination issues on Talk pages.
3. **Cross-reference**: Ingest a second document set (e.g., specifications after drawings). Revise existing pages by weaving new data into the right sections rather than appending. Cross-reference facts across document types. Update Talk pages.
4. **Construction docs**: Process RFIs, submittals, and field directives. Update affected area, equipment, and spec pages. Maintain the traceability chain via Pages Updated sections.
5. **Verification**: Run a verification cycle. Check each page against current document revisions. Flag stale information and identify pages needing updates.
6. **Review**: Final editorial review. Audit tone, completeness, and gaps. Produce a citation manifest pairing every factual claim with its source document.

A fresh MediaWiki instance is provisioned per run with templates and no content pages. The agent's conversation is resumed between checkpoints so it can build on prior work.

## Graders

Nine graders organized into three weighted tiers.

### Quality (50%)

**Reference** compares output against a human-written reference page. Measures section heading overlap, infobox field coverage, citation hash density, and category overlap. Each dimension is weighted and combined into a single score.

**Accuracy** resolves the citation manifest produced at the review checkpoint. For each claim, it checks whether the cited source document actually supports it. Claims are classified as supported, unsupported, or fabricated. Fabrications carry a 2x penalty.

### Content (30%)

**Completeness** checks structural elements: lead paragraph, infobox, body sections, `References` and `Bibliography` sections, categories, prose word count, subsection count, inline citations, and cross-references. Thresholds scale with checkpoint so earlier checkpoints have lower requirements.

**Editorial** is an LLM-graded evaluation of encyclopedic editorial conventions: neutral third-person tone, section organization, prose quality, wikitext syntax, and wikilink usage. Uses multi-pass grading with median scoring.

**Integration** measures revision quality when a page is updated across checkpoints. Checks whether new facts land in the right existing sections, whether the page reads as a unified article after revision, and whether conflicting data is noted on the Talk page. Only scored when the page content has changed since the previous checkpoint.

**Source criticism** evaluates whether document analysis includes critical assessment beyond raw extraction: cross-reference verification, gap identification, conflict detection, and coordination issues. LLM-graded.

### Mechanics (20%)

**Citations** validates citation templates (`{{Cite drawing}}`, `{{Cite spec}}`, `{{Cite rfi}}`, `{{Cite vault}}`, etc.) for required fields. Each template type has specific requirements: `hash` and `date` for document sources, `sheet` for drawings, `section` for specs. Score is valid citations over total.

**Cross-referencing** measures whether the page combines information from multiple document types. Counts facts that synthesize data across drawings, specs, and construction documents versus possible cross-references. Only scored when two or more document types are present.

**Tool usage** checks harness logs for expected CLI calls: `wai ingest`, `wai read`, `wai write`/`create`/`edit`. Binary per tool.

### Composite

```
page_composite = 0.5 * quality + 0.3 * content + 0.2 * mechanics
```

Within each tier, weight is split equally among the graders that produced a score. If an entire tier is absent, its weight redistributes proportionally to the tiers that are present.

The overall run composite splits 20% document analysis pages (averaged) and 80% content pages. Content pages are weighted: area 50% + equipment 30% + talk 20% when present.
