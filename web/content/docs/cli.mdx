The `wai` command-line tool is the primary interface for interacting with your ProjectWiki instance. It handles page editing, document ingestion, task management, and backup operations.

## Global flags

These flags are available on every command:

- `-j, --json` — output as JSON
- `-q, --quiet` — suppress non-essential output
- `-h, --help` — show help
- `-V, --version` — show version

## Pages

### `wai read`

Read a wiki page.

```bash
wai read <title>
wai read "Area 03 — Primary Clarifiers" --raw
wai read "Area 03 — Primary Clarifiers" --section 2
wai read "Area 03 — Primary Clarifiers" --offset 10 --limit 50
```

**Flags**:
- `-s, --section <n>` — read only section number n
- `--offset <n>` — start output from line n
- `-n, --limit <n>` — limit output to n lines
- `--raw` — output raw wikitext without formatting

### `wai write`

Write (overwrite) an entire page.

```bash
wai write "Area 03 — Primary Clarifiers" notes.txt
wai write "Area 03 — Primary Clarifiers" -c "Page content here"
wai write "Area 03 — Primary Clarifiers" -f notes.txt -m "Import notes"
echo "content" | wai write "Area 03 — Primary Clarifiers"
```

**Flags**:
- `-c, --content <text>` — provide content inline
- `-f, --file <path>` — read content from a file
- `-m, --summary <text>` — edit summary

Content can be provided via `-c`, `-f`, a positional file argument, or stdin.

### `wai edit`

Edit a page using find-and-replace.

```bash
wai edit "Area 03 — Primary Clarifiers" --old "old text" --new "new text"
wai edit "Spec:03 30 00" --old "typo" --new "fixed" --replace-all
wai edit "Drawing:C-301" --old "old" --new "new" --dry-run
```

**Flags**:
- `--old <text>` — text to find (required)
- `--new <text>` — replacement text (required)
- `--replace-all` — replace all occurrences (default: first only)
- `--dry-run` — preview changes without applying
- `-m, --summary <text>` — edit summary

### `wai create`

Create a new page.

```bash
wai create "New Page" -c "Page content"
wai create "Drawing:M-401" -f draft.txt -m "Initial analysis"
echo "content" | wai create "Issue:004"
```

**Flags**:
- `-c, --content <text>` — provide content inline
- `-f, --file <path>` — read content from a file
- `-m, --summary <text>` — edit summary

### `wai search`

Full-text search across all pages.

```bash
wai search "clarifier"
wai search "reinforcement" --limit 20
```

**Flags**:
- `-n, --limit <n>` — maximum number of results (default: 10)

### `wai upload`

Upload a file to the wiki.

```bash
wai upload drawing.pdf
wai upload drawing.pdf --name "C-301_Rev2.pdf"
wai upload photo.jpg -d "Site photo of clarifier excavation" -m "Upload site photo"
```

**Flags**:
- `--name <wiki-name>` — override the filename on the wiki
- `-d, --description <text>` — file description
- `-m, --comment <text>` — upload comment

## Sections

### `wai section list`

List all sections of a page.

```bash
wai section list "Area 03 — Primary Clarifiers"
```

### `wai section read`

Read a specific section of a page.

```bash
wai section read "Area 03 — Primary Clarifiers" 2
wai section read "Spec:03 30 00" 3 --raw
```

**Flags**:
- `--offset <n>` — start output from line n
- `-n, --limit <n>` — limit output to n lines
- `--raw` — output raw wikitext without formatting

### `wai section update`

Update a specific section of a page.

```bash
wai section update "Area 03 — Primary Clarifiers" 2 -c "New section content"
wai section update "Spec:03 30 00" 3 -f section.txt -m "Update section"
```

**Flags**:
- `-c, --content <text>` — provide content inline
- `-f, --file <path>` — read content from a file
- `-m, --summary <text>` — edit summary

## Talk pages

### `wai talk read`

Read a page's talk page.

```bash
wai talk read "Area 03 — Primary Clarifiers"
wai talk read "Area 03 — Primary Clarifiers" --thread "Clarifier mechanism vendor unknown"
wai talk read "Drawing:C-301" --raw
```

**Flags**:
- `--thread <name>` — read a specific thread
- `--raw` — output raw wikitext without formatting

### `wai talk create`

Create a new talk page thread.

```bash
wai talk create "Area 03 — Primary Clarifiers" -s "Missing dewatering spec" -c "No specification section covers dewatering requirements for this area."
```

**Flags**:
- `-s, --subject <text>` — thread subject (required)
- `-c, --content <text>` — provide content inline
- `-f, --file <path>` — read content from a file
- `-m, --summary <text>` — edit summary

## Discovery

### `wai link`

Show incoming and outgoing links for a page.

```bash
wai link "Area 03 — Primary Clarifiers"
wai link "PS-301A" --direction out
wai link "Spec:03 30 00" --direction in
```

**Flags**:
- `--direction <in|out|both>` — filter link direction (default: both)

### `wai category`

List all categories, or list pages within a specific category.

```bash
wai category
wai category "Process Areas"
```

### `wai changes`

Show recent changes across the wiki.

```bash
wai changes
wai changes --limit 20
```

**Flags**:
- `-n, --limit <n>` — maximum number of results (default: 10)

### `wai issue list`

List issues by status.

```bash
wai issue list
wai issue list --status open
wai issue list --status resolved
```

**Flags**:
- `-s, --status <status>` — filter by status: `open`, `resolved` (default: open)
- `-n, --limit <n>` — maximum number of results

### `wai issue read`

Read the details of a specific issue.

```bash
wai issue read 003
```

## Ingestion

### `wai ingest volume`

Ingest a directory of contract documents into the vault.

```bash
wai ingest volume ~/project/drawings/civil/ --type drawings --name "Volume 3 — Civil"
wai ingest volume ~/project/specs/ --type specs --name "Project Specifications"
wai ingest volume ~/project/geotech/ --type reports --name "Geotechnical Investigation"
```

**Flags**:
- `--type <type>` — document type: `drawings`, `specs`, `reports`, `submittals`
- `--name <name>` — volume name for the wiki index
- `--dry-run` — preview what would be stored without writing

### `wai ingest analyze`

Queue a previously ingested volume for agent analysis.

```bash
wai ingest analyze "Volume 3 — Civil"
wai ingest analyze "Project Specifications"
```

### `wai verify`

Run a verification cycle to check wiki content against current document revisions.

```bash
wai verify --all
wai verify "Area 03 — Primary Clarifiers"
```

**Flags**:
- `--all` — verify all pages
- `-n, --limit <n>` — maximum number of pages to verify

## Task queue

Agents use the task queue to coordinate work. These commands can also be used directly.

### `wai task list`

List tasks in the queue.

```bash
wai task list
wai task list --status pending
wai task list --status in-progress
wai task list --status done
wai task list --status failed
```

**Flags**:
- `-s, --status <status>` — filter by status: `pending`, `in-progress`, `done`, `failed` (default: pending)
- `-n, --limit <n>` — maximum number of results

### `wai task read`

Read the details of a specific task.

```bash
wai task read <id>
```

### `wai task create`

Create a new task.

```bash
wai task create -m "Analyze civil drawings for Area 03"
wai task create -m "Process RFI-042 and update affected pages"
```

**Flags**:
- `-m, --message <text>` — task description (required)
- `--source <name>` — associate the task with a document volume

### `wai task claim`

Claim a pending task for processing.

```bash
wai task claim <id>
```

### `wai task complete`

Mark an in-progress task as completed.

```bash
wai task complete <id>
wai task complete <id> -m "Created area page and 4 drawing pages"
```

**Flags**:
- `-m, --message <text>` — output message to record

### `wai task fail`

Mark an in-progress task as failed.

```bash
wai task fail <id>
wai task fail <id> -m "Missing structural drawings for this area"
```

**Flags**:
- `-m, --message <text>` — failure reason to record

### `wai task requeue`

Return a failed or in-progress task to the pending queue.

```bash
wai task requeue <id>
```

## Backup

### `wai export`

Export a full wiki backup as a tar archive.

```bash
wai export ~/backups
wai export ~/backups --dry-run
```

**Flags**:
- `--dry-run` — preview what would be exported without writing

### `wai import`

Import from a backup archive.

```bash
wai import ~/backups/projectwiki-2026-01-15.tar
wai import backup.tar --force
wai import backup.tar --dry-run
```

**Flags**:
- `--force` — overwrite existing data
- `--dry-run` — preview backup contents without restoring

## Auth

### `wai auth login`

Store wiki credentials. Prompts interactively for any missing values.

```bash
wai auth login
wai auth login --server http://localhost:8080 --username admin --password secret
```

**Flags**:
- `--server <url>` — wiki server URL
- `--username <name>` — wiki username
- `--password <pass>` — wiki password

### `wai auth logout`

Remove stored credentials.

```bash
wai auth logout
```

### `wai auth status`

Show connection status and stored credential info.

```bash
wai auth status
```

## Update

### `wai update`

Update the CLI to the latest version.

```bash
wai update
```

The CLI also checks for updates automatically in the background once every 24 hours and prints a notice to stderr if a newer version is available.
