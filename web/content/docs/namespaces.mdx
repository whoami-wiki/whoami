---
title: Namespaces
description: "How the construction project wiki uses MediaWiki namespaces"
---

Namespaces separate different kinds of content so each page has a clear purpose. The wiki inherits the namespace system from MediaWiki and adds five custom namespaces for construction project management.

A namespace appears as a prefix before the page title. Pages in the Main namespace have no prefix. All other namespaces use the format `Namespace:Page Title`.

## Main

The default namespace. Area pages, equipment pages, system pages, and project-level content live here with no prefix.

```
Area 03 — Primary Clarifiers
PS-301A
Project Standards
Submittal Log
```

Area pages document process areas with discipline-organized sections. Equipment pages document individual tagged items. The Project Standards page captures Division 01 requirements that configure agent behavior.

## Talk

MediaWiki's built-in discussion namespace. Every Main page automatically gets a corresponding Talk page at `Talk:Page Title`.

```
Talk:Area 03 — Primary Clarifiers
Talk:PS-301A
```

Talk pages serve as the verification and coordination record for each page. They hold verification status, active gaps, resolved items, coordination issues, document history, and agent logs.

### Standard structure

Talk pages use the following sections as needed, in this order:

1. **Verification status** — `{{Verification}}` template with status (complete, in-progress, not-started), last verified date, and source documents
2. **Active gaps** — open questions marked `{{Open}}` that need resolution
3. **Resolved** — closed items marked `{{Closed}}`, corrected ones `{{Superseded}}`
4. **Coordination issues** — cross-discipline coordination items
5. **Document history** — chronological record of which documents contributed to this page
6. **Agent log** — one entry per task: ID, date, what changed, link to task page

### Active gaps

```wikitext
=== Clarifier mechanism vendor unknown ===
{{Open}}
Spec 46 33 00 lists performance requirements but no basis-of-design
manufacturer. Will be resolved when submittal is received.
```

### Resolved

```wikitext
=== Wall rebar spacing at penetrations ===
{{Superseded}}
Previously documented as #5 at 12" O.C. per Drawing S-302.

{{Closed}}
RFI-042 response (2026-01-22): S-305 Detail 7 governs. Use #5 at 8" O.C.
for penetrations > 6" diameter. Area page and Spec 03 30 00 updated.
```

## Drawing

A custom namespace (ID 100) for drawing analysis pages. Each contract drawing gets its own page following the observations-before-interpretation protocol.

```
Drawing:C-301
Drawing:S-302
Drawing:M-401
```

Drawing pages include a title block table, physical observations, dimensions and elevations, cross-references to other drawings, and engineering interpretation.

## Spec

A custom namespace (ID 102) for specification section pages. Each CSI MasterFormat section gets its own page preserving the Part 1/2/3 structure.

```
Spec:03 30 00
Spec:43 21 00
Spec:01 10 00
```

Spec pages use `{{Verbatim}}` blocks for exact contract language and maintain an Active Modifications table tracking changes from RFIs and other construction documents.

## Construction

A custom namespace (ID 104) for construction-phase documents: RFIs, submittals, field directives, change orders, and serial letters.

```
Construction:RFI-042
Construction:SUB-033000-001
Construction:FD-015
```

Each page includes a type-specific infobox and a Pages Updated section documenting which wiki pages were modified as a result of this document.

## Issue

A custom namespace (ID 106) for project-level issues: drawing conflicts, spec ambiguities, missing documents, and risk items.

```
Issue:001
Issue:003
Issue:017
```

Issues track severity, status, and recommended actions. They are distinct from talk page Active Gaps — issues are project-level problems requiring action, while gaps are page-level uncertainties.

## Task

A custom namespace (ID 108) for agent work logs. Each task page documents what an agent did, which pages it created or modified, and the analysis performed.

```
Task:0003
Task:0008
```

Tasks follow a lifecycle: pending → in-progress → done/failed. Failed tasks can be requeued.

## Summary

| Namespace | Prefix | ID | Purpose |
| --- | --- | --- | --- |
| Main | (none) | 0 | Area pages, equipment, project content |
| Talk | `Talk:` | 1 | Verification status, gaps, coordination |
| Drawing | `Drawing:` | 100 | Drawing analysis pages |
| Spec | `Spec:` | 102 | Specification section pages |
| Construction | `Construction:` | 104 | RFIs, submittals, field directives |
| Issue | `Issue:` | 106 | Conflicts, ambiguities, risk items |
| Task | `Task:` | 108 | Agent work logs |

For page type conventions, see [Page Types](/docs/page-types).
