---
title: Setting Up Your First Project
description: "Go from contract documents to a populated project wiki"
---

This guide walks you through setting up a construction project wiki from scratch — ingesting contract documents, building the project framework, and letting agents analyze drawings and specifications into structured wiki pages.

You don't need to spell out formatting rules, citation templates, or page conventions in your prompts. The [extension](/docs/installation#step-3-install-the-extension-for-your-agent-harness) handles all of that automatically.

## Step 1: Set up the project

Launch the desktop app and complete the setup wizard. Choose construction project mode and enter your project name. The wizard creates:

- **Main Page** with a project dashboard
- **Project Standards** stub page
- **Submittal Log** page
- **Drawing Index** page

## Step 2: Ingest Division 01 specifications

Start with the general requirements specs. These configure how the agent handles submittals, RFIs, and quality control for the rest of the project.

```
Ingest the Division 01 specifications from ~/project/specs/division-01/
and populate the Project Standards page.
```

The agent reads Division 01 specs (Summary of Work, Submittals, RFI Procedures, Quality Control) and populates the Project Standards page with document precedence, submittal procedures, and RFI procedures.

## Step 3: Ingest drawings

Split your drawing volumes into individual pages, then point the agent at them:

```bash
# Split a PDF volume
pdftk volume3-civil.pdf burst output pages/page_%03d.pdf

# Catalog and queue for analysis
wai ingest volume pages/ --type drawings --name "Volume 3 — Civil"
wai ingest analyze "Volume 3 — Civil"
```

The agent analyzes each drawing following the structured protocol: title block extraction, physical observations, dimensional analysis, cross-reference verification, and engineering interpretation.

## Step 4: Ingest specifications

```bash
wai ingest volume specs/ --type specs --name "Project Specifications"
wai ingest analyze "Project Specifications"
```

The agent extracts CSI Part 1/2/3 structure, preserves verbatim contract language, identifies submittal requirements, and pre-populates the Submittal Log.

## Step 5: Process construction documents

As the project progresses, feed in RFIs, submittals, and field directives:

```
Process RFI-042 regarding clarifier wall reinforcement at penetrations.
The response from the engineer is attached.
```

The agent creates the Construction document page, traces back to affected specifications and drawings, and updates all wiki pages that need to reflect the change — maintaining the traceability chain.

## Step 6: Review issues

As agents analyze documents, they identify conflicts, ambiguities, and gaps:

```
wai issue list --status open
wai issue read 003
```

Review open issues and decide which need RFIs, which need design clarification, and which can be resolved with existing information.

## Step 7: Verify

Run periodic verification cycles to ensure wiki content matches current document revisions:

```
wai verify --all
```

The agent checks each page's verification status, flags stale information, and identifies pages that need updating based on new documents.
