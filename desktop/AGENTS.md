# desktop/

The Electron app that bundles MediaWiki + PHP + Lua + ffmpeg into a single
double-clickable installer. This was the project's original wiki host —
when an agent runs `wai write`, the CLI talks to the local MediaWiki
instance running inside this app.

## Status: being phased out

The new `frontend/` package (Next.js) is the active path going forward.
Reasons:

- MediaWiki is wikitext-shaped; the project has migrated to markdown.
- Bundling PHP + a stack of native tools per-platform is brittle.
- The Next.js renderer is faster, more flexible, and easier to extend
  for features like the family tree browser.

The desktop app still works and is what users currently install. Don't
delete it. But:

- **Avoid building new features here** — put them in `frontend/`.
- **Bug fixes are fine** — keep the existing user base working until
  the migration completes.
- **Treat the bundling scripts as load-bearing** — `scripts/build-php.sh`,
  `scripts/bundle-mediawiki.sh`, `scripts/bundle-lua.sh`, and
  `scripts/bundle-ffmpeg.sh` produce the binary blobs the app needs at
  runtime; they're not random shell scripts.

## Layout

| Path                   | Purpose                                                              |
| ---------------------- | -------------------------------------------------------------------- |
| `src/main.ts`          | Electron main process. Boots PHP server, starts MediaWiki.           |
| `src/php-server.ts`    | Spawns and supervises the bundled PHP CLI server.                    |
| `src/setup.ts`         | First-run setup: extract bundled MediaWiki, write LocalSettings.     |
| `src/tray.ts`          | macOS tray icon + menu.                                              |
| `src/updater.ts`       | electron-updater wiring for auto-update.                             |
| `renderer/`            | The thin renderer process (Vite-built). Most of the UI is MediaWiki itself. |
| `resources/`           | Bundled assets (templates, MediaWiki extensions, theme CSS).         |
| `scripts/bundle-*.sh`  | Per-platform builders for PHP, MediaWiki, Lua, ffmpeg blobs.         |

## Build and run

```bash
npm run dev                  # boot the app in development
npm run bundle-deps          # rebuild PHP/MediaWiki/Lua/ffmpeg blobs
npm run build                # tsc + vite build (renderer)
npm run dist                 # full electron-builder package
npm run reset                # nuke local app data (~/Library/Application Support/whoami/data)
```

`bundle-deps` is slow. You only need to re-run it when the bundled
runtime versions change.

## Pitfalls

- **Editing `LocalSettings.php` from the IDE** — the file is
  templated/generated by `setup.ts` on first run. Hand-edits get
  clobbered. Update the template under `resources/templates/`.
- **Bumping MediaWiki without testing migrations** — pages are stored
  in MediaWiki's database (or now: the markdown files in
  `~/whoami/pages/`). Schema changes affect existing users.
- **Forgetting to commit a release** — desktop releases follow the
  `release: desktop-v1.0.5` convention and trigger user auto-update.
